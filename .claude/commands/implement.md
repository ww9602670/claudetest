# 逐步实现（Implement）

基于任务清单，逐条执行 Task 并实现代码。

## 执行步骤

1. 读取 `specs/<功能名>/tasks.md`（功能名从 $ARGUMENTS 获取）
2. 读取 `specs/<功能名>/spec.md` 和 `specs/<功能名>/design.md` 作为上下文
3. 找到第一个未完成的 Task（即 `[ ]`），开始执行
4. 按照 Task 中描述的操作步骤逐一实现
5. 实现完成后，按 Task 的验证方式进行自验证
6. 验证通过后，将该 Task 的 `[ ]` 改为 `[x]`，并在其后追加完成记录
7. 向用户汇报当前进度，然后继续下一个 Task
8. 如果遇到需要用户决策的问题，暂停并提问

## 每个 Task 完成后的记录格式

在该 Task 条目下追加：
```
  - ✅ 完成时间：YYYY-MM-DD HH:mm
  - 实际改动：列出实际修改/新增的文件
  - 备注：（如有偏差或额外说明）
```

## 执行原则

- **一次只做一个 Task**：不要并行执行多个 Task
- **严格按顺序**：除非前置 Task 被阻塞，否则不跳过
- **忠于设计**：实现必须与 design.md 保持一致；如果发现设计有问题，先告知用户并更新设计文档
- **最小改动**：只做 Task 要求的改动，不要顺手重构无关代码
- **及时汇报**：每完成一个 Task 输出进度摘要

## 进度汇报格式

```
📋 进度更新
━━━━━━━━━━━
✅ 已完成：Task X.X - <标题>
📂 改动文件：file1, file2
🔜 下一步：Task X.X - <标题>
📊 总进度：M/N（已完成/总数）
```

## 注意事项

- 如果 tasks.md 不存在，提示用户先执行 `/project:tasks`
- 如果所有 Task 已完成，提示用户执行 `/project:verify`
- 每次执行此命令会从上次中断的地方继续（查找第一个 `[ ]`）
