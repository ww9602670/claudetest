### P2 + P4 — gate-write.js 扩展门禁覆盖范围

**修改文件**：`.claude/hooks/gate-write.js`

**原逻辑（有漏洞）**：
- 仅检查路径中包含 `/features/` 的写入
- `features/` 以外的业务代码（如 `src/auth/`）完全不受门禁

**新逻辑**：
- 引入**豁免路径白名单**：`.claude/`、`docs/`、`specs/`、`features/` 下的写入直接放行（治理配置与文档区）
- 所有**非豁免路径**（业务代码，如 `src/`、`lib/` 等）均须通过门禁：
  1. 必须存在处于 `approved / implementing / verifying` 状态的活跃 feature
  2. 目标路径须在该 feature 的 `allowed_paths` 内
  3. 目标路径不得命中 `forbidden_paths`
- 相对路径计算使用 `process.cwd()` 做基准，兼容 Windows 盘符路径
