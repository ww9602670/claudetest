  人工验收建议检查的 5 个重点

  1. gate-write.js 实际拦截效果：在 example-login 的 status 改为 draft 后，尝试 Edit 一个 src/ 下的文件，确认 hook
  能真正 exit 2 拦截。这是整个治理体系的核心闸门。
  2. gate-bash.js 白名单模式：将某个 feature 的 status 设为 verifying，执行一个不在 required_checks 中的命令（如 git
  log），确认被拦截；再执行 npm run lint 确认放行。
  3. feature.json 模板字段完整性：对比模板 (docs/spec-system/模板/AI执行版/feature.json) 与示例
  (features/example-login/feature.json)，确认字段一致且无遗漏（当前一致）。
  4. 5 个 Skill 能否被 Claude Code 正确识别：在 Claude Code 中输入 /spec、/design 等命令，确认能被正确路由到对应的
  SKILL.md 而非旧的 .claude/commands/ 下的文件。
  5. spec-workflow.md 的文档互锁是否生效：在一个新 feature 目录中，跳过 spec.md 直接尝试创建 design.md，确认系统（或
  AI）能识别并拒绝这一违规操作。
