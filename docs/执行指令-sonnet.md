# Claude 执行指令（Sonnet 4.6 版）

## 一、你的身份与职责

你当前使用 **Sonnet 4.6**。

你的职责不是重新审计和重新设计，而是：

- 读取交接文件
- 进行第一阶段最小落地施工
- 尽量用最少次数 patch 完成文件创建与修正
- 完成后立即停止，等待审核

---

## 二、本轮唯一目标

本轮只做一件事：

**严格依据交接文件，完成第一阶段最小落地施工。**

---

## 三、优先读取顺序

你必须优先只读取：

1. `docs/handoff/phase1-plan.json`

只有在交接文件信息不足时，才允许补充读取：

2. `docs/需求.md`
3. `docs/技术难点与解决方案.md`

要求：

- 不要重新全量审计项目
- 不要重新读取大量无关文件
- 不要主动扩展读取范围
- 如果你认为交接文件不够，必须先说明“缺什么”，再最小化补读

---

## 四、本轮只允许执行的内容

你只允许创建或修正以下第一阶段内容：

1. 最小 `CLAUDE.md`
2. 核心 rules
3. 2 个 gate hooks
4. 小白版 4 份模板
5. AI 执行版 4 份模板
6. `feature.json` 模板
7. 1 个最小示例
8. 迁移说明
9. 基本验收清单

---

## 五、严格禁止事项

本轮禁止：

1. 禁止进入第二阶段
2. 禁止修改业务代码
3. 禁止重新设计治理体系
4. 禁止创建完整自动修复循环
5. 禁止创建 deploy / release 能力
6. 禁止创建完整 agents 集合
7. 禁止大量补建兼容 commands
8. 禁止重新引入 `/project:*`
9. 禁止重新引入 `feature.yaml`
10. 禁止把 `specs/` 当作执行态目录
11. 禁止新增嵌套 `CLAUDE.md`
12. 禁止新增嵌套 `.claude/rules/`
13. 禁止新增嵌套 `.claude/skills/`
14. 禁止把治理状态写入 auto memory 作为依据
15. 禁止输出冗长的过程解释

---

## 六、你必须遵守的关键硬约束

施工过程中，必须遵守以下规则：

### 1. 执行态目录
- 只能使用 `features/<feature-id>/`

### 2. 命令体系
- 只允许：
  - `/spec`
  - `/design`
  - `/tasks`
  - `/implement`
  - `/verify`

### 3. 状态文件
- 只允许 `feature.json`

### 4. 状态机
必须固定为：

- `draft -> review -> approved -> implementing -> verifying -> done | blocked`

### 5. hooks 覆盖范围
写入门禁必须覆盖：

- `Edit`
- `MultiEdit`
- `Write`

### 6. Windows 约束
- `command_windows` 为权威字段
- `command_bash` 仅为兼容字段
- 默认优先 Node.js
- 不得生成 bash-only 主路径

### 7. 隐藏规则源
- 只能审计
- 不能作为新的治理事实源
- 不能作为新的执行入口

### 8. 阶段确认闸门
- 第一阶段完成后不得自动进入第二阶段
- 阶段切换必须人工确认

---

## 七、施工方式要求

你必须按以下方式执行：

1. 先输出：
   - 本轮将创建哪些文件
   - 本轮将修改哪些文件
   - 本轮禁止动哪些内容

2. 然后尽量一次性完成最少次数 patch

3. 完成后立即停止，不得继续扩展

要求：

- 不要边想边做很多次小 patch
- 不要顺手修与第一阶段无关的内容
- 如果发现交接文件与项目现状冲突，先停下并报告冲突点

---

## 八、完成后的输出格式

本轮完成后，你只输出：

1. 新增文件清单
2. 修改文件清单
3. 未完成项
4. 当前是否已具备第一阶段最小闭环
5. 等待 Opus 审核

不要继续做第二阶段，不要继续扩展。

---

## 九、成功标准

只有同时满足以下条件，才算本轮成功：

1. 你只读取了最小必要文件
2. 你没有重新全量审计项目
3. 你只完成了第一阶段允许创建的内容
4. 你没有修改业务代码
5. 你没有进入第二阶段
6. 你没有引入被禁止的路径、命令或状态文件格式
7. 你完成后立即停止
