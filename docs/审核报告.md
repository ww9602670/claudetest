  结论
  这版 需求.md 已经可以直接驱动 Claude 去做，而且大概率能把“第一阶段最小落地”实现对。相比前几版，主要分叉点基本都被你锁
  住了。

  但如果你的标准是“直接交给 Claude，就要稳定、少歧义、门禁不能被钻空子”，我建议再补 4 条硬约束后再用：

  - hook 写入门禁必须覆盖 MultiEdit
  - 审计必须覆盖 CLAUDE.local.md、.claude/settings.local.json 和 CLAUDE.md 的 @import
  - allowed_paths / forbidden_paths 必须锁死匹配语义
  - command_windows 必须定义为权威字段，command_bash 只作兼容

  如果你还想让 Claude 严格分阶段停下来等你确认，再补“阶段确认闸门”这一条。
