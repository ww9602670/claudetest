# 阶段需求 03：第一阶段可审闭环增强

## 阶段目标

把“已经能跑的第一阶段骨架”增强到“可审、可追踪、可复验”的状态。

本阶段不再修最核心的门禁漏洞，而是补齐审计闭环，让后续模型和人都能看得见执行证据。

## 前置条件

必须先完成：

- `docs/spec-system/阶段需求/02-第一阶段余量收尾与执行基线增强.md`

## 本阶段输入

只读取以下内容：

- 第一阶段已修复后的 `.claude/hooks/*`
- `.claude/skills/**/SKILL.md`
- `.claude/agents/*.md`
- `docs/spec-system/验收清单.md`
- `docs/spec-system/验收清单审查报告.md`
- `features/example-login/**`
- `tests/verify-hooks.ps1`

## 本阶段必须完成

1. 增加证据沉淀机制
   - 补入 `PostToolUse(Edit|MultiEdit|Write)` 或等价记录能力
   - 补入 `PostToolUse(Bash)` 或等价检查摘要能力

2. 统一 evidence 目录规范
   - 至少明确：
     - `change-summary.md`
     - `checks.md`
     - `iteration-log.md`

3. 补齐 skill 元信息
   - 按第一阶段实际需要，补充：
     - `argument-hint`
     - 必要的 `allowed-tools`
     - 必要的 `paths`
     - 必要的 `user-invocable`

4. 清理遗留 agent 的旧路径假设
   - 不允许继续把 `specs/<功能名>/...` 作为默认输出路径
   - 若暂时保留 agent，也要明确其兼容边界

5. 增加“已加载 / 已注册 / 已触发”的验证方案
   - 至少形成一份正式验证步骤
   - 让后续审查不再只靠人工猜测

## 允许修改范围

只允许修改：

- `.claude/hooks/*`
- `.claude/skills/**/SKILL.md`
- `.claude/agents/*.md`
- `docs/spec-system/**`
- `features/example-login/**`
- `tests/**`

## 禁止事项

- 禁止引入 CI
- 禁止引入 Git 门禁
- 禁止实现 `Stop` / `SubagentStop`
- 禁止扩展为完整第二阶段 agent 编排
- 禁止触碰 deploy / release

## 本阶段产出

至少应产出：

1. 标准化 evidence 目录规则
2. 证据沉淀 hook 或等价脚本
3. 元信息补齐后的 skill 文件
4. 清理后的 agent 文档
5. 一份正式“加载验证”说明或脚本

## 完成判定

以下条件同时满足才算完成：

1. 不依赖口头解释即可看懂执行痕迹
2. evidence 路径和文件名不再自由发挥
3. skill 和 agent 的边界对其他模型是清晰的
4. 后续审查可以基于仓库内证据复验，而不是只看会话结果

## 需要人工介入的点

1. 是否要求 `example-login` 升级为真实可运行演练
2. 是否继续保留现有 3 个 agent，还是在下一阶段统一重建
