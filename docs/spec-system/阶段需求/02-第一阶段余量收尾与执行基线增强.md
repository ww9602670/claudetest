# 阶段需求 02：第一阶段余量收尾与执行基线增强

执行状态：**已完成**

## 阶段目标

由于原定的最小闭环修复已经在阶段 01 的实际执行中提前完成并被正式吸收，本阶段不再重复 gate 修复。

本阶段只处理**剩余的第一阶段收尾工作**，目标是让后续模型能够在更低噪音下继续推进，而不必重新判断执行基线是否可靠。

必须先完成：

- `docs/spec-system/阶段需求/01-现状收敛与口径统一.md`
- `docs/spec-system/阶段需求/01-阶段审查报告.md`

- `tests/verify-hooks.ps1`

## 本阶段必须完成

1. 固化“已加载 / 已注册 / 已触发”的正式验证方案
   - 至少形成一份可直接执行的验证说明
   - 优先产出最小验证说明、验证步骤或轻量验证脚本，不扩展为重型自动化验证框架
   - 让后续审查不再依赖会话口头解释

2. 收口执行侧遗留路径假设
   - 清理 `.claude/agents/*.md` 中仍把 `specs/<功能名>/...` 当默认输出路径的内容
   - 仅保留明确的兼容说明，不允许继续作为当前执行路径假设

3. 补齐第一阶段最低 skill 元信息
   - 按实际需要补充：
     - `argument-hint`
     - 必要的 `allowed-tools`
     - 必要的 `paths`
     - 必要的 `user-invocable`
   - 目标是降低误触发和误读，不追求第二阶段的复杂拆分

4. 固化占位示例与最小 evidence 基线
   - 让 `example-login` 的 demo-only 属性在所有相关说明中保持一致
   - 明确第一阶段最小 evidence 产物要求
   - 但本阶段不引入 `PostToolUse` 自动沉淀

5. 产出一份阶段 02 审查或验收文档
   - 明确哪些仍未做
   - 明确下一阶段的唯一输入基线
