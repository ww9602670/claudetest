## 八、required_checks 的执行要求

你不能假设某个脚本天然存在。
你必须按以下原则建立：

1. 先建立最基础、最真实可执行的检查
2. 能从现实中跑通的，优先
3. 暂时不存在的脚本，不得先写进 checks 再假装它存在
4. 如果需要某个脚本，先创建，再纳入 checks
5. 每条 check 必须有对应 evidence_file
6. 每条 check 必须能解释“它为什么是当前阶段合理的真实工程动作”
