## 使用规则

1. 一次只激活 **1 份阶段需求文档**
2. 默认按编号顺序执行：`01 -> 02 -> 03 -> 04 -> 05`
3. 未满足上一个阶段的完成判定，不得跳到下一份
4. 如需知道为什么这样拆分，再回看：
   - `docs/需求.md`
   - `docs/spec-system/验收清单审查报告.md`
5. 正常执行时，不需要把上述长文档整份重新灌入上下文
