## 我们打算怎么做？

（用非技术语言描述技术方案，让非开发人员也能理解）

示例：我们会在登录页面增加一个表单，用户输入邮箱和密码后，系统会去数据库核对信息。如果正确，就在用户的浏览器里保存一个"通行证"，7 天内都不需要重新登录。

## 会改动哪些地方？

（列出会修改或新增的文件/页面/功能，用非技术语言描述）

- 示例：登录页面（新增）
- 示例：导航栏（修改，增加"登录"和"退出"按钮）
- 示例：数据库（修改，增加保存"通行证"的表）

## 不会改动哪些地方？

（明确声明不会碰的部分）

- 示例：注册页面不会改动
- 示例：现有用户的数据不会变动
- 示例：其他页面的功能不受影响

## 可能的风险？

（列出可能出现的问题和对应处理方式）

- 示例：如果用户忘记密码怎么办？→ 本功能不处理，密码找回是单独功能
