# 需求规格 — 网页版贪食蛇游戏

> AI 执行版 | feature-id: snake-game
> 
> **目标**: 验证 spec 驱动开发机制是否成立

---

## 功能概述

实现一个经典的网页版贪食蛇游戏，玩家通过键盘方向键控制蛇移动，吃到食物后蛇身变长并得分，撞墙或撞到自己则游戏结束。

**核心价值**: 作为04阶段首个正式实施对象，用于验证 spec 三件套驱动开发、非程序员说明同步产出、自动开发/修复、evidence 与复审机制是否完整可行。

---

## 需求范围

### In Scope（包含）

- FR-01: 游戏区域显示 - 显示游戏画布和得分
- FR-02: 蛇的移动 - 支持上下左右四个方向控制
- FR-03: 食物系统 - 随机生成食物，蛇吃到食物后身体变长
- FR-04: 计分系统 - 显示当前得分
- FR-05: 碰撞检测 - 撞墙或撞到自己时游戏结束
- FR-06: 重新开始 - 游戏结束后可以重新开始
- FR-07: 速度控制 - 随着得分增加，蛇的移动速度逐渐加快
- NFR-01: 响应性 - 按键响应延迟 < 100ms
- NFR-02: 兼容性 - 支持主流现代浏览器（Chrome/Firefox/Edge）

### Out Scope（不包含）

- 多人游戏模式
- 排行榜功能
- 音效和背景音乐
- 移动端触屏控制
- 复杂的视觉特效
- 关卡系统

---

## 输入/输出规格

### 输入

| 操作 | 类型 | 触发方式 | 说明 |
|------|------|---------|------|
| 方向控制 | 方向键 | 键盘输入 | 上下左右四个方向 |
| 重新开始 | 鼠标点击 | 点击按钮 | 游戏结束后显示重新开始按钮 |

### 输出

| 场景 | 输出内容 |
|------|---------|
| 游戏进行中 | 显示蛇、食物、当前得分 |
| 吃到食物 | 蛇身增长、得分+10、速度提升 |
| 游戏结束 | 显示"游戏结束"、最终得分、重新开始按钮 |

---

## 成功标准

- AC-01: 打开页面后能看到游戏区域和初始得分（0分）
- AC-02: 按方向键能控制蛇的移动方向
- AC-03: 蛇能吃到食物，吃到后得分增加10分
- AC-04: 蛇吃到食物后身体变长
- AC-05: 撞墙时游戏结束并显示最终得分
- AC-06: 撞到自己时游戏结束并显示最终得分
- AC-07: 游戏结束后可以点击重新开始按钮重新游戏
- AC-08: 随着得分增加，蛇的移动速度有明显的提升
- AC-09: 页面加载时间 < 2秒
- AC-10: 按键响应延迟 < 100ms

---

## 依赖与约束

### 技术约束
- 必须使用纯前端技术实现（HTML/CSS/JavaScript）
- 不依赖后端服务或数据库
- 单页面实现，无需路由
- 所有代码在一个HTML文件或同目录下的少数文件中

### 设计约束
- 界面简洁清晰，适合非程序员理解和测试
- 游戏规则符合经典贪食蛇玩法
- 代码结构清晰，易于维护和扩展

### 禁止修改
- 不修改现有其他 feature 的代码
- 不修改仓库根目录的配置文件
- 不影响现有 CI/CD 流程

## 验收优先级

| 优先级 | 功能项 | 说明 |
|--------|--------|------|
| P0 | 基本移动、吃食物、碰撞检测 | 核心游戏逻辑 |
| P1 | 计分系统、重新开始 | 完整游戏体验 |
| P2 | 速度随得分提升 | 增强游戏性 |

**注意**: 本次任务的首要目标是验证机制，而非实现完美游戏功能。P0 功能完整且机制验证通过即可判定完成，P1/P2 功能可作为后续迭代。
