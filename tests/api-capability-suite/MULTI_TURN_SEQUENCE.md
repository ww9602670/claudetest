# 多轮记忆测试提示词

这部分需要你手动逐轮发送，不能合并成一条消息，否则不能验证真正的跨轮记忆。

建议在自动化执行结束后，同一个 Claude 会话里继续做。

## 发送顺序

### 第 1 轮

```text
请记住这 4 个标记，只回复“已记住”，不要重复内容：
amber-17
lake-42
orbit-89
cedar-24
```

期望：

- 只回复“已记住”或等价简短确认
- 不应把 4 个标记全部回显出来

### 第 2 轮

```text
请运行一个 shell 命令，列出 tests/api-capability-suite 目录下有多少个文件和文件夹。只告诉我数字和一句简短说明，不要提刚才的标记。
```

期望：

- 成功调用工具
- 不应混入第 1 轮标记

### 第 3 轮

```text
现在只输出第 2 个和第 4 个标记，格式必须是 X|Y。
```

期望答案：

`lake-42|cedar-24`

### 第 4 轮

```text
把这 4 个标记按数字从小到大排序，每行一个，并把结果写入 tests/api-capability-suite/runs/<RunLabel>/logs/multi_turn_sorted.txt。
```

期望顺序：

1. `amber-17`
2. `cedar-24`
3. `lake-42`
4. `orbit-89`

### 第 5 轮

```text
第 1 轮里我明确要求你不要做什么？
```

期望答案：

- 不要把 4 个标记全部重复出来

### 第 6 轮

```text
请用一句话判断这组多轮记忆测试是否通过，并把判断和原因追加到 tests/api-capability-suite/runs/<RunLabel>/logs/multi_turn_observations.md。
```

## 判定建议

- 全部命中且无编造：通过
- 轻微格式偏差但核心记忆正确：谨慎通过
- 任一标记错误、遗漏、顺序混乱明显：不通过
